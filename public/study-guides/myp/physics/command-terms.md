# MYP Physics — Command Terms Reference

This page covers every command term seen across the 18 Physics e-assessment sessions (M16–M25, N16–N24). For each term you will find: its definition, which criterion it typically appears under, how many marks it usually carries, what the examiner expects, and the most common mistakes students make.

## How Mark Schemes Apply Each Term

### Calculate

**What the mark scheme awards marks for:**
1. Correct formula written (or implied by correct substitution)
2. Correct substitution of values with units
3. Correct numerical answer
4. Correct unit on the answer

Marks are often split as: 1 mark (formula/working) + 1 mark (correct answer) + 1 mark (unit), but the exact split varies.

**ECF applies:** If you substitute a wrong value from a previous part, you can still earn the calculation marks in the current part if the method is correct.

**Unit mark is independent:** Even if your number is wrong, you earn the unit mark if you write the correct unit for the quantity being calculated.

**Common patterns:**
- Multi-step calculations (e.g. find resistance then find power): each step earns marks separately
- "Show that" questions: you must reach the given answer; if you don't, show all working for partial credit
- Rearranging a formula: show the rearranged formula before substituting

### Design / Formulate (Criterion B)

Criterion B questions follow a predictable structure. Each component is marked separately.

**Research question**
- Must identify the independent variable (IV) and dependent variable (DV)
- Must be phrased as a question ("How does X affect Y?")
- Specific and measurable: "How does the length of a wire affect its resistance?" not "How does wire affect electricity?"

**Hypothesis**
- Must use the If/Then/Because structure (or equivalent)
- If: state what you will change (IV)
- Then: state what you predict will happen to the DV and the direction of change
- Because: give a physics reason (a law, principle, or mechanism)
- Example: "If the length of the wire increases, then the resistance will increase, because there are more collisions between electrons and the lattice ions over a longer path."

**Variables**
- Independent variable (IV): what you deliberately change; must be specific
- Dependent variable (DV): what you measure; must be specific, including units
- Control variables (CV): at least 2–3; must state what they are AND why controlling each one matters
- Avoid vague CVs like "everything else" — name them specifically

**Method**
- Must be a numbered list of steps (or clearly structured paragraphs)
- Must specify equipment with sizes/quantities where relevant
- Must explain how the IV is changed in steps
- Must explain how the DV is measured (which instrument, how many readings, how repeated)
- Must include sufficient data: state how many different values of the IV you will test and how many repeats

**Sufficient data statement**
- State the range of IV values AND the number of repeats
- Example: "Test 5 different lengths (10 cm, 20 cm, 30 cm, 40 cm, 50 cm) and repeat each measurement 3 times."

### Discuss / Evaluate (Criterion D)

Criterion D extended-response questions require structured paragraphs, not bullet points (unless the question specifically asks for a list).

**What a high-scoring Discuss answer includes:**
1. A brief introduction linking the topic to the real-world context given
2. At least two arguments/viewpoints on each side (for and against, or positive and negative impacts)
3. Evidence or data from the passage/stimulus to support each point
4. A concluding statement that makes a judgment

**What a high-scoring Evaluate answer includes:**
1. A clear statement of what is being evaluated (the claim, method, or technology)
2. Specific strengths with physics reasons
3. Specific weaknesses or limitations with physics reasons
4. A conclusion that weighs the evidence and makes a judgment

**Common mark scheme language for Criterion D:**
- "Accept any reasonable point that is supported by physics reasoning"
- "Award 1 mark for point + 1 mark for explanation/development"
- "[max 6]" — write more than 6 good points but only 6 will be credited; prioritise your strongest points

## Common Mistakes by Command Term

| Command term | Most common mistake | How to avoid it |
|-------------|-------------------|----------------|
| **Calculate** | Forgetting the unit on the final answer | Write the unit immediately after the number; check if a separate unit mark exists |
| **Calculate** | Not showing the formula before substituting | Always write the formula first; "V = IR" before "V = 2 × 5" |
| **Calculate** | Using the wrong formula | Memorise the formula triangle; check what quantity the question asks for |
| **Describe** | Vague language ("it gets bigger") | Use specific physics language ("the resistance increases") and quantify if possible |
| **Design** | Identifying only IV and DV, omitting control variables | Criterion B always asks for CVs; list at least 3 with reasons |
| **Design** | Vague method ("measure the result each time") | Specify the instrument ("use an ammeter in series"), the readings ("record 3 repeats"), and the range |
| **Discuss** | Only giving one side of the argument | Examiners expect balanced responses; plan both sides before writing |
| **Evaluate** | Saying "the experiment was good" without evidence | Every evaluative claim must be supported by specific data or a physics reason |
| **Explain** | Stating what happens but not why | The word "because" is almost always needed; link cause to effect explicitly |
| **Formulate** | Writing a hypothesis without a physics reason | The "Because" clause must name a physical law or mechanism, not just repeat the prediction |
| **Identify** | Writing a sentence when a word or phrase is needed | Read the mark allocation; 1-mark "identify" questions need only the answer, not an explanation |
| **Justify** | Giving a reason without linking it to the choice | Name the choice first ("I would use a thermistor"), then give the specific reason |
| **Plot** | Plotting points inaccurately or using the wrong scale | Use the full axis; check that each plotted point matches the data table |
| **Plot** | Drawing a curved line of best fit through all points | A straight line of best fit is correct for linear relationships; not all points need to be on it |
| **State** | Over-explaining when just a term or value is needed | Match your answer length to the mark; 1-mark "state" = one item |
| **Suggest** | Writing "I don't know" or leaving it blank | Any physically plausible suggestion with a reason is accepted (WTTE); attempt it |
| **Verify** | Calculating a value but not comparing it to the given value | Verification requires the comparison: "My value of X matches/is close to the given value of Y" |

## Quick Reference: Command Term + Criterion Pairings

Use this to predict what a question is asking based on criterion and command term.

| If you see... | Criterion | Expect to... |
|-------------|-----------|-------------|
| Calculate, State, Label, Identify | A | Recall and apply knowledge; show working for calculations |
| Explain, Describe, Outline | A, D | Write in sentences; link cause to effect |
| Compare, Discuss, Evaluate, Justify | D | Write a structured paragraph; address multiple aspects |
| Formulate, Design, Identify (variables) | B | Follow the investigation design structure precisely |
| Plot, Determine, Organise, Evaluate (conclusion) | C | Work with data; draw graphs; check your answer against expected values |
| Suggest (improvements) | C, D | Propose specific, actionable changes; give a reason for each |

## Mark Scheme Conventions (Quick Reference)

| Term | Meaning |
|------|---------|
| **WTTE** | Words to that effect — any phrasing with the correct meaning is accepted |
| **ORA** | Or reverse argument — the opposite direction of reasoning is also credited |
| **ECF** | Error carried forward — consistent use of a wrong answer earns subsequent marks |
| **"seen or implied"** | The mark is awarded if the working makes the value obvious, even if not stated |
| **[max N]** | Maximum N marks from a longer list of possible marking points |
| **Dependent mark** | This mark can only be awarded if the previous mark was earned |
| **All correct** | Drag-drop / select-all: every item must be correct; no partial credit |
| **Unit mark** | Awarded independently of the numerical mark |
| **"judge by eye"** | Diagrams are assessed holistically; precise measurement is not required |

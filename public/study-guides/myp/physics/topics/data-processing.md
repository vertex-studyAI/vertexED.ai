# Data Processing (Criterion C)

---

## 1. Key Concepts

### Organising Raw Data into Tables

Raw data collected from an experiment must be presented in a clear, structured table before any processing takes place. The mark scheme treats this as a multi-part skill: correct column headers, units placed in the header only (not repeated in every cell), data in a consistent number of decimal places, and rows ordered in ascending or descending order of the independent variable.

Units belong in the column header alongside the quantity name — written as, for example, "Distance / m" or "Tension / N". Values in the body of the table must never carry units of their own. Decimal places must be consistent throughout a column: if one value is 0.43, every value in that column must be given to two decimal places.

**M17 Q7b (5 marks) — Organise and Present (sail/trolley experiment):**
The mark scheme awarded one mark each for: both headers correct; units in headers matching the data; units only in headers (not in cells); values in order of increasing or decreasing radius; values to a consistent number of decimal places. Missing any single element cost a mark.

> **Mark scheme callout:** "Values have been recorded to a consistent number of decimal places" is an independent mark. You can have correct headers and ordering and still lose this mark if, for example, one row shows 5.70 and another shows 8.1 rather than 8.10.

---

### Calculating Means and Averages

When multiple trials are collected for a single value of the independent variable, the mean must be calculated and presented to the same number of decimal places as the trial data. The mark scheme checks that the arithmetic is correct and that the result is rounded consistently — not truncated.

Units in the data must be consistent before averaging. If trial values are given in mixed units (e.g. one in metres, one in centimetres, one in scientific notation), they must all be converted to the same unit first.

**N18 Q5b (2 marks) — Calculate average bounce height:**
Three repetitions were given as 0.52 m, 0.47 cm, and 5.5 × 10⁻¹ cm. The first mark required evidence of correct unit conversion (0.47 cm → 0.0047 m; 5.5 × 10⁻¹ cm → 0.0055 m). The second mark awarded the correct average: 0.51 m. No ECF was allowed from an incorrect conversion.

> **Mark scheme callout:** Mixed units in raw data are a deliberate trap. Always convert everything to the same unit before averaging. The conversion step is itself a marking point.

---

### Significant Figures and Decimal Places

Two distinct concepts are tested. Significant figures (sf) are required when asked explicitly — usually for calculations. Decimal places (dp) govern consistency within a table column. Confusing the two is penalised.

When a question says "give your answer to two significant figures," the final answer must be rounded, not just truncated. The mark for rounding is independent of the calculation mark, so a correctly rounded wrong answer can still earn the rounding mark (but only if method is shown).

**N19 Q5a (3 marks) — Speed of sound (echo):**
The mark scheme gave three separate marks: (1) noting the total distance is 220 m (double the 110 m wall distance); (2) using speed = distance/time; (3) rounding the answer correctly to 2 sf: 280 m s⁻¹. The rounding mark was explicitly stated as available even if the numerical answer was incorrect, provided the rounding itself was done correctly.

**M24 Q1d (2 marks) — Energy released by burning coal:**
Answer required to 2 sf. Full marks: 1.1 × 10¹³ J. The mark scheme noted "award 2 marks for a correct answer rounded to 2 sf." Giving 1.125 × 10¹³ without rounding scored only 1 mark.

> **Mark scheme callout:** "Rounded correctly to 2 sf" is often a separate, independent mark. Show the unrounded value then round explicitly so the examiner can award both marks.

---

### Plotting Graphs

Plotting questions award marks for: labelling both axes with quantity and unit (e.g. "Bounce height / m"); choosing a scale that uses more than half the available grid space; plotting points accurately; and drawing a line of best fit that has roughly equal numbers of points above and below the line.

The line of best fit is judged by eye — it does not need to pass through any specific point, including the origin. Points should be plotted as small crosses or dots, not blobs. Axis labels must distinguish between the two variables — writing "height" alone on both axes does not earn the mark.

**N18 Q4e (1 mark) — Draw line of best fit (bounce height vs pressure):**
The mark scheme stated simply: "line must go through all of the data points." This was an unusual case because the data was tightly linear. The standard requirement — roughly equal distribution above and below — applies in almost all other questions.

**N23 Q4g (2 marks) — Choose the better graph and justify:**
Graph B was the correct choice because data points covered more than half the plotting area and allowed a more accurate trend line. Graph A was penalised because its scale was too compressed. No mark was awarded for selecting Graph A.

> **Mark scheme callout:** Scale matters. A graph where data occupies less than half the grid fails the scale mark. Always extend axes to fill the grid, not just to the next round number above the largest value.

---

### Reading Values from Graphs — Gradient, Intercept, Extrapolation

Gradient questions require two points taken from the line of best fit (not from data points), separated by a large enough interval. The mark scheme typically requires working to be shown (two coordinates stated, difference calculated) and the unit to be included separately. The gradient mark is awarded "consistent with their line of best fit," so candidates who draw a correct line but calculate the gradient from data points rather than from the line are penalised.

Intercept reading is an independent skill. The y-intercept is where the line crosses x = 0; its significance must be interpreted in context, not just stated numerically.

**M22 Q4h (3 marks) — Calculate gradient of frequency vs 1/4L graph:**
Three marks: (1) working shown with points separated by more than 1000 Hz in the y-direction; (2) correct value 343 ± 10; (3) unit m s⁻¹. The unit mark was awarded independently. "Consistent with their line of best fit" was explicitly stated — a candidate with a slightly different LOBF earned full marks if their gradient calculation matched their own line.

**N18 Q4g (3 marks) — Explain y-intercept significance (bounce vs pressure):**
Three marks: (1) stating the y-intercept value (~0.40 m); (2) noting the ball is deflated at 0 kPa; (3) concluding it will not bounce in reality. The mark scheme penalised answers that just restated the numerical intercept without interpreting its physical meaning.

> **Mark scheme callout:** "Two points from the line of best fit" means from the line — not from data points. Points must be widely separated to minimise rounding error. Always state both coordinates and show the subtraction.

---

### Identifying Anomalies and Outliers

An anomalous result is one that lies noticeably far from the line of best fit. The mark scheme expects candidates to identify which data point is anomalous by reference to the graph (stating the x or y coordinate), and to state the direction of the error — whether the recorded value is too high or too low relative to the trend.

**N19 Q5f (2 marks) — Identify anomalous data point (echo experiment):**
Two marks: (1) identification of the data point at 400 m or approximately 1 s; (2) stating the time is too low — the actual time should be greater to follow the trend and be closer to the line of best fit. A candidate who said "the point is wrong" without identifying direction earned only 1 mark.

> **Mark scheme callout:** Always state both which point is anomalous (by coordinate) and which direction the error lies (too high / too low). "There is an anomaly" alone scores zero.

---

### Evaluating Validity — Commenting on Trends and Hypothesis Support

Evaluating a hypothesis requires three elements: stating whether the trend matches the prediction (direction), determining whether the mathematical relationship matches (e.g. proportional vs linear vs inverse), and drawing a conclusion about whether the hypothesis is supported or not. The mark for the conclusion is conditional — it is only awarded if the prior evidence has been correctly cited.

Proportionality means the line of best fit passes through the origin AND is straight. A straight line that does not pass through the origin shows a linear but not proportional relationship. This distinction is tested repeatedly.

**N18 Q4h (2 marks) — Evaluate hypothesis (bounce height proportional to pressure):**
The mark scheme awarded: (1) if the relationship were proportional, the line would pass through the origin; (2) the line does not pass through the origin, so the relationship is not proportional. A candidate who said "the hypothesis is supported because bounce height increases with pressure" scored zero — the direction of trend alone does not confirm proportionality.

**N23 Q4h (3 marks) — Evaluate prediction (drop height and bounce height proportional):**
Three marks: (1) noting the trendline is curved or that doubling drop height does not double bounce height; (2) the increase is not proportional; (3) the prediction is therefore incorrect. The third mark was conditional on the first two being awarded.

> **Mark scheme callout:** "The hypothesis is supported" without citing data earns zero. Always quote two data points or refer to the graph explicitly. "Proportional" requires the line to pass through the origin — never accept a straight line as evidence of proportionality without checking the intercept.

---

### Uncertainty, Error Sources, and Their Effects

Error questions ask for a named source of error (specific, not generic) and a correctly linked description of how that error affects the result. "Human error" is not accepted — the error must be named (e.g. "heating of the wire changes its resistance"). The effect must be directional where possible — does the error increase or decrease the calculated value?

**M24 Q3h (2 marks) — Suggest source of error in resistance investigation:**
Two marks: (1) a specific source, such as wires heating up, fluctuating readings, or probes not placed at the exact measured length; (2) the correctly linked effect — e.g. heating increases resistance, or calculated resistance is not for the correct length. The mark scheme explicitly excluded "increase the number of trials" as an answer.

> **Mark scheme callout:** "Increase the number of trials" is never an acceptable error source — it is a method improvement. Name a physical cause (friction, heating, parallax, air resistance) and link it to the specific effect on the result.

---

### Percentage Calculations — Efficiency and Percentage Change

Efficiency = (useful output / total input) × 100. The mark scheme awards the formula application, the numerical answer, and sometimes the unit (% or none) as separate marks. Percentage change follows the same pattern. Both types require the correct denominator — using the wrong base value (e.g. using output instead of input) is a common error that costs the calculation mark.

**N23 Q3d (3 marks) — Compare efficiency of LED vs filament bulb:**
Three separate marks: (1) efficiency of filament bulb = 5%; (2) efficiency of LED = 0.02 / 0.05 = 40%; (3) LED efficiency is higher. The third mark was conditional on both prior marks being awarded. Candidates who calculated only one efficiency and drew a comparison earned at most 2 marks.

**M24 Q1e (2 marks) — Useful power output at 28% efficiency:**
The efficiency formula must be shown or implied. 26 700 × 0.28 = 7476 MW. Two marks: (1) applying the efficiency relationship; (2) correct numerical answer with a consistent unit.

> **Mark scheme callout:** For efficiency: always use total input as the denominator, not output. For percentage change: always divide by the original value. Show the formula substituted with numbers before giving the final answer.

---

### Constructing a Data Table from Scratch

Some questions ask candidates to construct a full table to record planned data — not just to present existing results. These tables must include column headers with units, the specific values of the independent variable that will be used (typically five), spaces for at least three trials, and a column for calculated averages or processed data.

**M24 Q4g (4 marks) — Construct data table for magnetic launcher investigation:**
Four marks, one each for: column headers for initial height and maximum height including units; five increments of initial height listed; spaces for at least three trials; a column for average/mean maximum height. Units repeated in the data cells (not just the header) cost a mark.

> **Mark scheme callout:** A constructed table needs five values of the IV, three trial columns, and an average column. Missing any one of these costs a mark. Units go in the header only.

## 3. Mark Scheme Insights

### What the mark scheme rewards

- **Specific identification of errors:** A named physical cause (e.g. "the wire heats up as current flows, increasing its resistance") earns the mark. Generic statements ("there is human error") earn nothing.
- **Directional anomaly statements:** "The data point at 400 m is too low — the actual time should be greater" earns both marks. "There is an anomaly" earns zero.
- **Showing gradient working:** Two coordinates from the line (not from data points), separated by a large interval, with subtraction shown. Unit stated independently.
- **Proportionality checked against the origin:** Candidates who verify that the line passes through (0, 0) AND is straight earn the proportionality mark. Candidates who only note the line is straight do not.
- **Consistent decimal places in tables:** Every value in a column must match. One cell at 0.43 and another at 0.4 costs a mark.
- **Units in headers only:** Units written in table cells rather than the header cost a mark (N18 Q5c, M22 Q5c, M24 Q4g — all penalise this).
- **Mixed-unit conversion shown:** In N18 Q5b, the conversion of cm to m was a marking point independent of the final average.
- **Evaluative conclusion linked to evidence:** "The hypothesis is incorrect because the line does not pass through the origin (as shown at coordinates...)" earns the conclusion mark. "The hypothesis is incorrect" without evidence does not.
- **Intercept interpreted in context:** In N18 Q4g, stating the y-intercept value was only the first mark. Interpreting it (ball is deflated, will not bounce) was needed for marks 2 and 3.
- **Error Carried Forward (ECF):** If a plotted point is wrong but the line of best fit is correctly drawn through it, the LOBF mark is still awarded. Gradient calculated consistently with a candidate's own line is credited.

### What the mark scheme penalises

- **"Increase the number of trials" as an error source:** Explicitly excluded in M24 Q3h. This is a method improvement, not an error source.
- **"Proportional" without checking the origin:** In M17 Q5i, the mark scheme explicitly states "do not accept proportional" — only "linear" is accepted for a straight-line trend unless the origin is verified.
- **Selecting data points (not line points) for gradient:** The gradient must be read from the line of best fit, not from the nearest data point.
- **Units in data cells:** Tables where units appear in the cell rather than the header lose the "units in header only" mark consistently across all sessions.
- **Selecting Graph A (compressed scale):** In N23 Q4g, no mark was available for choosing the compressed graph, regardless of justification given.
- **Vague hypothesis evaluation:** "The results support the hypothesis" with no data citation earns zero across all sessions. The mark for conclusion is consistently conditional on prior evidence marks.
- **"Human error" without specification:** Never credited. Must name the physical mechanism.
- **Averaging without converting units first:** In N18 Q5b, averaging 0.52 m + 0.47 cm + 5.5 × 10⁻¹ cm without conversion earned zero for both marks.
- **Rounding to decimal places instead of significant figures when asked for sf:** These are different operations. A question asking for 2 sf expects 280, not 279.46 or 279.5.

## 4. Common Mistakes

| Mistake | Correction |
|---------|------------|
| Writing units in every table cell (e.g. "0.43 m" in the cell) | Units belong in the column header only: "Bounce height / m" then bare numbers below |
| Saying "the line is straight so the relationship is proportional" | Proportional requires the line to pass through the origin AND be straight — always check the intercept |
| Averaging raw data in different units without converting first | Convert all values to the same unit, show the conversion, then calculate the mean |
| Reading gradient from a data point rather than from the line of best fit | Place a ruler on the drawn LOBF, pick two points widely spaced on the line, show coordinates and subtraction |
| Omitting the unit from a gradient calculation | Gradient unit = y-axis unit divided by x-axis unit — state it as a separate mark point |
| Identifying an anomaly without stating the direction of error | Always state: "the value is too high/low — it should be [direction] to follow the trend" |
| Writing "increase the number of trials" as a source of error | Name a physical cause: friction, heating, parallax, inconsistent drop angle — then link to the effect on the result |
| Saying "the hypothesis is supported/rejected" without citing data | Always quote graph coordinates, specific values, or an explicit check of the intercept/gradient |
| Using "proportional" and "linear" interchangeably | Linear = straight line (any position); proportional = straight line through the origin only |
| Giving averages or table data to inconsistent decimal places | Round all values in a column to the same dp before entering them in the table |
| Giving an unrounded answer when a specific sf is requested | Round explicitly; show the unrounded value first so both marks are visible to the examiner |
| Labelling graph axes as just "height" when there are two heights | Distinguish clearly: "Drop height / m" and "Bounce height / m" — "height" alone is not accepted |

## 5. Revision Priorities

1. Practise presenting raw data in tables: column headers with units, values to consistent decimal places, ascending order of IV. Use past table questions (M17 Q7b, N18 Q5c, M22 Q5c) to check against the mark scheme — every mark point is distinct.

2. Master the distinction between linear and proportional. For every graph you draw or read, check whether the line passes through the origin. If not, say "linear" — never "proportional."

3. Drill the gradient calculation procedure: pick two points from the line (not from data), widely spaced, write both coordinates, subtract, divide, state the unit. Practise on M22 Q4h and N20 Q6c.

4. Learn the anomaly identification formula: state the coordinate of the outlying point AND state whether the recorded value is too high or too low. N19 Q5f is the model answer.

5. Learn to name specific error sources and link them to directional effects. Keep a short list: wire heats up → resistance increases; ruler at an angle → length measurement is too large; parallax when reading a scale → value is over- or under-estimated. Practise M24 Q3h.

6. Know the three-part hypothesis evaluation structure: (1) state the observed trend; (2) compare the mathematical relationship (check origin); (3) state the conclusion — all three marks are conditional in sequence.

7. For efficiency and percentage calculations, always write the formula with numbers substituted before giving the final answer. Use total input (not output) as the denominator. Practise N23 Q3d and M24 Q1e.

8. Construct full data tables from memory: five IV values, three trial columns, one average column, units in headers only. M24 Q4g is the benchmark question.

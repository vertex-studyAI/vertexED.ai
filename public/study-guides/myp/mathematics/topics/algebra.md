## 1. Key Concepts

### Expanding and Factorising Expressions

Expanding means multiplying out brackets; factorising means reversing that process by writing an expression as a product of factors. Both skills appear frequently as steps inside larger questions — the mark scheme almost always awards a method mark for showing the expansion or factored form before reaching the final answer.

To expand a double bracket, multiply every term in the first bracket by every term in the second:

> (x − 5)(x + 1) = x² + x − 5x − 5 = x² − 4x − 5

To factorise a quadratic ax² + bx + c, find two numbers that multiply to ac and add to b:

> x² − 4x − 5 → need two numbers that multiply to −5 and add to −4 → (−5) and (+1) → (x − 5)(x + 1)

**Worked example (M21 Q3a/3b):** The line f(x) = 2x + 4 and the parabola g(x) = (x − 1)² − 2 intersect. Setting them equal:

```
2x + 4 = (x − 1)² − 2
2x + 4 = x² − 2x + 1 − 2
0 = x² − 4x − 5
```

Factorising: (x − 5)(x + 1) = 0, giving x = 5 and x = −1. Substituting back into f(x) gives points A = (−1, 2) and B = (5, 14).

> **Mark scheme note (M21 Q3b):** The mark scheme explicitly awards a separate mark (.1) for the correct factorisation or correct use of the quadratic formula. Jumping straight to x-values without showing the factored form risks losing this method mark.

### Solving Linear Equations and Inequalities

A linear equation is solved by isolating the unknown using inverse operations. Each step must be shown — mark schemes allocate one mark per correct algebraic step in multi-step problems.

**Sign rule for inequalities:** When multiplying or dividing both sides by a **negative** number, the inequality sign must be reversed. This is one of the most penalised errors in the exam.

> Solve: 3 − 2f(x) = 11
>
> Step 1: −2f(x) = 11 − 3 = 8
> Step 2: f(x) = 8 ÷ (−2) = −4
> Step 3: Read from the domain table → x = 0

**Worked example (M17 Q4c):** f(x) is defined on the domain {0, 2, 4, 6} with values {−4, 0, 2, 3}. Solve 3 − 2f(x) = 11.

The mark scheme allocates four separate marks:
- •1 One correct algebraic step: −2f(x) = 8
- •2 Dividing correctly: −f(x) = 4
- •3 f(x) = −4
- •4 x = 0 (from the table)

> **Mark scheme note (M17 Q4c):** The step "3 − 2f(x) − 11 = 0" does NOT earn the first method mark — the mark scheme explicitly rejects this rearrangement. The required first step is to move the constant: −2f(x) = 8.

### Simultaneous Equations

Two equations in two unknowns are solved by elimination (add or subtract the equations to remove one variable) or substitution (express one variable in terms of the other and substitute).

**Elimination method:**

> x + y = 22  …(1)
> x − y = 12  …(2)
>
> Add (1) and (2): 2x = 34, so x = 17
> Substitute into (1): 17 + y = 22, so y = 5

**Worked example (M18 Q2):** A picture shows frog + dragonfly = 22 and frog − dragonfly = 12. Find 2 frogs + 2 dragonflies.

Elimination gives x = 17, y = 5. Then 2(17) + 2(5) = 44 but the target expression is 2x + 2y = 2(x + y) = 2 × 22 = 44. The mark scheme awards six marks: one for setting up the system, one for the elimination step, one for each variable, one for doubling correctly, and one for the final sum of 54.

> **Mark scheme note (M18 Q2):** The answer 54 with no working earns all 6 marks. However, if working is shown, every step that is incorrect costs the corresponding mark — partial credit depends on tracing through "their" values from earlier steps.

### Sequences — Arithmetic and Geometric, nth Term

An **arithmetic sequence** has a constant common difference d. The nth term is:

> u_n = u_1 + (n − 1)d

A **geometric sequence** has a constant common ratio r. The nth term is:

> u_n = u_1 × r^(n−1)

To identify a sequence type, check first differences (arithmetic → constant) and second differences (quadratic → constant second difference, but not first).

**Worked example (M17 Q7a–c and M21 Q8a–c):** In M17 Q7, as k increases 1, 2, 3, 4, 5 the x-coordinate of point A follows: 1/2, 1/4, 1/6, 1/8, 1/10. The denominator increases by 2 each time, giving the general rule x_A = 1/(2k).

In M21 Q8, the longer base of a trapezium follows L = 4, 6, 8, 10, 12, 14 as n = 1, 2, 3, 4, 5, 6. The arithmetic sequence has first term 4 and common difference 2, giving L = 2n + 2.

> **Mark scheme note (M17 Q7c):** "1/2k" (ambiguous) is not accepted — the rule must be written unambiguously as 1/(2k) or (2k)⁻¹. Notation matters for full marks.

> **Mark scheme note (M21 Q8b):** "Arithmetic" alone does not earn the mark. The description must state the specific difference (adds 2, increases by 2, etc.) to be accepted.

### Indices and Surds

**Index laws (key rules):**

| Law | Form |
|-----|------|
| Product | a^m × a^n = a^(m+n) |
| Quotient | a^m ÷ a^n = a^(m−n) |
| Power of a power | (a^m)^n = a^(mn) |
| Negative index | a^(−n) = 1/a^n |
| Fractional index | a^(1/n) = ⁿ√a |
| Zero index | a^0 = 1 |

**Surd rules:**

> √a × √b = √(ab)
> √a ÷ √b = √(a/b)
> k√a + m√a = (k+m)√a  (can only add like surds)

Simplify √20: √(4 × 5) = 2√5. Similarly √45 = √(9 × 5) = 3√5.

**Worked example (M21 Q2a/2b):** Addition and multiplication grids with surds and index expressions.

For the surd addition grid: √20 + √5 = 2√5 + √5 = 3√5. For the index multiplication grid: (7/2)a × 2b = 7ab; (7/2)a × 4ab⁻² = 14a²b⁻².

> **Mark scheme note (M21 Q2b):** Answers must be in simplified index form. If a result like 7ab is written as 7a¹b¹ without simplification, the mark scheme may deduct one mark for non-simplified form. Follow the instruction "simplified exact form" exactly.

### Rearranging Formulae

Rearranging means making a different variable the subject. Use the same inverse-operation logic as solving equations, but treat all other letters as constants.

**Worked example:** Make r the subject of V = (4/3)πr³:

> V = (4/3)πr³
> 3V = 4πr³
> r³ = 3V/(4π)
> r = ∛(3V/(4π))

This type of rearrangement underpins several geometry questions — for example, in M17 Q5c and M21 Q5b, volume formulas are equated and rearranged to find an unknown length.

> **Mark scheme note:** In "show that" questions the working must flow forward to the given answer (AG — answer given). Rearranging backwards from the answer earns no marks.

### Substitution and Evaluation

Substitution means replacing a variable with a given value and computing the result. In function notation: evaluate f(4) means put x = 4 into the rule for f.

**Worked example (M17 Q4a/4b):** f(4) = 2 (read from the table). Then 3f(4) − 1 = 3(2) − 1 = 5.

For the formula w = 24(100)^(−r) in M21 Q7e: substitute r = 0.77:

> w = 24 × (100)^(−0.77) = 24 × 0.02884... = 0.6922... ≈ 0.69 (2 s.f.)

> **Mark scheme note (M21 Q7e):** The mark scheme awards a separate mark for the correct substitution step before the numerical evaluation. Writing the substitution explicitly as 24(100)^(−0.77) earns the method mark even if arithmetic is wrong.

### Linear Programming

Linear programming optimises an objective function subject to linear constraints. The feasible region is bounded by the constraint inequalities, and the optimal value occurs at a vertex of that region.

**Key steps:**
1. Write each constraint as a linear inequality.
2. Identify the feasible (shaded) region from the graph.
3. Evaluate the objective function at each vertex.
4. State the vertex that maximises (or minimises) the objective.

**Worked example (M18 Q3a–d):** x = cookies, y = muffins. Constraints:

| Inequality | Meaning |
|------------|---------|
| x + y ≤ 500 | Total cannot exceed 500 |
| x ≥ 100 | Minimum 100 cookies |
| y ≥ 50 | Minimum 50 muffins |
| y ≤ x | Cannot sell more muffins than cookies |

Feasible region vertices: A(100, 100), B(100, 50), C(450, 50), D(250, 250).

Profit P = x + 1.5y evaluated at each vertex:

| Vertex | P |
|--------|---|
| A(100, 100) | 250 |
| B(100, 50) | 175 |
| C(450, 50) | 525 |
| D(250, 250) | 625 |

Maximum profit = 625 CAD at D(250, 250).

> **Mark scheme note (M18 Q3b):** Interpreting the coefficient 1.5 as "a muffin is 1.5 times the profit of a cookie" without stating the actual profit value per item does NOT earn the mark. The required answer is that the coefficient represents the profit (in CAD) per muffin, i.e. $1.50 per muffin.

> **Mark scheme note (M18 Q3a):** Using strict inequalities (< instead of ≤) when the boundary line is included loses the mark. Read the diagram carefully to decide whether boundary points are included.

## 3. Mark Scheme Insights

### What the mark scheme rewards

- **Showing each algebraic step separately.** In multi-step equation solving (e.g. M17 Q4c, 4 marks), each mark corresponds to one operation. Writing only the answer earns zero; writing each rearrangement step earns each mark in sequence.
- **Unambiguous notation.** General rules for sequences must be written in correct mathematical notation: 1/(2k) not 1/2k; L = 2n + 2 not just "two times n plus two."
- **Matching to predicted or given values when verifying.** The final step in a "Verify" question always requires an explicit statement that the calculated value equals the value from the pattern or table (M17 Q7d, M21 Q8d).
- **Context in constraint interpretation.** In linear programming, the mark scheme requires the constraint to be described in terms of the real-life scenario (cookies, muffins, etc.), not just symbolically.
- **Correct rounding with stated degree of accuracy.** When asked to round (e.g. M21 Q7e: 2 significant figures), the mark scheme gives a separate mark for the rounded answer. State the rounding explicitly.
- **Forward working only in "Show that" questions.** Starting from the given answer and working backwards earns no marks (M21 Q3a).

### What the mark scheme penalises / does not accept

- **Rearranging to "expression = 0" as first step.** Writing 3 − 2f(x) − 11 = 0 instead of −2f(x) = 8 in M17 Q4c earns 0 for the first mark.
- **Vague sequence descriptions.** "Arithmetic," "linear," "increasing," or "the even numbers" alone are rejected — the description must specify the actual difference or rule (M21 Q8b).
- **Ambiguous fraction notation.** 1/2k is ambiguous (could mean (1/2)k); the mark scheme requires 1/(2k) or the equivalent (M17 Q7c).
- **Forgetting to reverse inequality when dividing by a negative.** This is penalised in any question involving inequalities with negative coefficients.
- **Coefficient interpretation without context.** In M18 Q3b, "1.5 is the profit of a muffin" needs to state that it means $1.50 profit per muffin — a ratio comparison (muffin is 1.5× cookie) alone is rejected.
- **Non-simplified index or surd answers.** M21 Q2b specifies "simplified index form"; leaving an answer as 7a¹b¹ instead of 7ab may cost a mark.
- **ECF (error carried forward) only applies with working shown.** In M17 Q4b, if f(4) ≠ 2, ECF on the expression 3f(4) − 1 is only allowed if working is visible.

## 4. Common Mistakes

| Mistake | Correction |
|---------|-----------|
| Forgetting to reverse the inequality sign when dividing by a negative | Whenever both sides are divided (or multiplied) by a negative number, flip ≤ to ≥ (or < to >) |
| Sign errors when expanding brackets, especially with a negative in front | (x − 1)² = x² − 2x + 1, NOT x² − 1; distribute the negative carefully |
| Writing the general rule in ambiguous notation | Write 1/(2k) not 1/2k; the mark scheme rejects ambiguous forms |
| Testing a verification with a value already in the table | "Verify" requires n ≥ 5 (a value outside the original data); testing n ≤ 4 earns only a special case mark |
| Identifying constraints with the wrong inequality direction | Sketch the feasible region; check which side of each boundary line is shaded before writing ≤ or ≥ |
| Solving "3 − 2x = 11" as "2x = 11 − 3" (dropping the negative) | The correct first step is −2x = 11 − 3 = 8, so x = −4, not x = 4 |
| Confusing arithmetic and geometric sequences | Arithmetic: check first differences are constant; geometric: check ratios are constant |
| Stopping at factorised form without finding coordinates | After solving x² − 4x − 5 = 0, substitute x = 5 and x = −1 back into either equation to get the y-coordinates |
| Using strict inequalities when the boundary is included | A solid boundary line means ≤ or ≥; a dashed boundary means < or > |
| Interpreting the coefficient in a profit formula as a ratio | State the profit per unit item, not the ratio between coefficients |

## 5. Revision Priorities

1. Practise multi-step equation solving with explicit steps — write one operation per line and check each step earns its mark under the mark scheme.
2. Drill the inequality sign-reversal rule: make and memorise a short example (e.g. −2x < 6 → x > −3) so it becomes automatic.
3. Learn to identify arithmetic vs geometric sequences: compute first differences; if constant, arithmetic; if ratios are constant, geometric; if second differences are constant, the rule is quadratic.
4. Practise writing nth term rules in unambiguous notation and verify them with a value outside the given data.
5. For simultaneous equations, be fluent in both elimination and substitution — know which is faster given the form of the equations.
6. For linear programming questions, always: (a) read the feasible region carefully, (b) list all vertices, (c) evaluate the objective function at every vertex before stating the optimum.
7. Memorise the six index laws and the surd simplification rules (√(ab) = √a × √b; add like surds only) — these appear in grids and multi-step questions.
8. In "Show that" and "Verify" questions, always work forward and explicitly state that the two values match at the end — the final comparison statement is worth its own mark.
